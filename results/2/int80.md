Взаємодія з операційною системою через переривання int 0x80

Що таке int 0x80?
Це програмне переривання в архітектурі x86, яке дозволяє програмі користувача звернутися до ядра Linux для виконання системного виклику. Фактично, це сигнал процесору переключитися в режим ядра, щоб виконати певну дію (вивід тексту, читання файлу тощо).

Як це працює?
Програма кладе номер необхідної функції в регістр eax, а параметри — в інші регістри (ebx, ecx, edx). Після цього викликається команда int 0x80.

Найбільш вживані функції та їх параметри:

sys_exit (eax = 1) — Завершує роботу програми.

ebx: код завершення (0 — успішне виконання).

sys_read (eax = 3) — Зчитує дані з джерела.

ebx: дескриптор файлу (0 для введення з клавіатури).

ecx: адреса буфера (пам'яті), куди записуються дані.

edx: кількість байтів для зчитування.

sys_write (eax = 4) — Виводить дані на екран або у файл.

ebx: дескриптор файлу (1 для виведення на екран).

ecx: адреса рядка тексту в пам'яті.

edx: довжина рядка в байтах.

sys_open (eax = 5) — Відкриває файл.

ebx: адреса імені файлу в пам'яті.

ecx: режим доступу (читання, запис тощо).
